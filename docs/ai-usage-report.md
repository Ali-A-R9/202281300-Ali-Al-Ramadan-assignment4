# AI Usage Report – Assignment 4

## 1. Tools Used and Use Cases

### ChatGPT
I used ChatGPT as a support tool during development, debugging, documentation writing, and final polishing of the portfolio.

Main uses:
- reviewing my portfolio against the assignment rubric
- debugging JavaScript logic and validating expected behavior
- improving project structure and documentation
- refining UI details such as layout ideas, section organization, and wording
- checking whether the portfolio met Assignment 4 expectations
- helping improve error messages and overall user experience
- helping polish the final submission files and presentation preparation

## 2. Benefits of Using AI

Using AI tools helped me in several ways:

- I solved bugs faster than working completely alone
- I got feedback on code structure and readability
- I improved my understanding of JavaScript logic, especially filtering, sorting, and state handling
- I refined the documentation more professionally
- I generated ideas for small design improvements without having to redesign the whole project
- AI helped me compare my work against the rubric and identify missing pieces before submission

---

## 3. Challenges and Limitations

There were also limitations when using AI tools:

- some suggestions were too generic and did not fully match my portfolio style
- some generated code needed simplification to fit the course level and my project structure
- a few UI suggestions looked less natural than my original design, so I had to reject them
- AI did not always understand the exact assignment context unless I explained it clearly
- I had to verify that every suggestion actually worked in my own files

Because of this, I could not simply accept AI output directly. I had to test, adapt, and sometimes rewrite parts manually.

---

## 4. Learning Outcomes

Using AI during this assignment improved my learning in both technical and workflow areas.

### Technical learning
I improved my understanding of:
- API usage with `fetch()`
- asynchronous JavaScript and promises
- DOM updates and reusable rendering functions
- client-side validation
- `localStorage` for persistent UI state
- responsive UI thinking and component organization

### Workflow learning
I also learned how to:
- ask better technical questions
- compare AI suggestions instead of following the first answer
- review generated code critically
- use AI as a support tool rather than as a shortcut
- connect assignment requirements with implementation decisions more clearly

---

## 5. Responsible Use and Modifications

I used AI responsibly and made sure the final submission reflects my own understanding.

Steps I followed:
- reviewed all AI suggestions before using them
- modified generated code to fit my own portfolio structure
- tested features in the browser after making changes
- rejected ideas that did not match the assignment or my design
- rewrote parts that felt too complex, unnecessary, or not original enough
- kept transparency by documenting the use of AI in this report and in the README

This helped me maintain academic integrity while still benefiting from AI-assisted support.

## 6. Examples of AI-Supported Work

Examples of tasks where AI was useful:
- checking form validation rules and feedback flow

- suggesting small visual improvements like the skills section styling and icon integration

## 7. Final Reflection

AI was useful in this assignment because it acted like a development assistant, reviewer, and idea generator.

However, the final result still depended on my own decisions:
- I chose what to keep
- I changed what did not fit
- I tested the final behavior
- I made sure I understood the final code and documentation

Overall, AI improved my efficiency and helped me reflect more carefully on quality, usability, and documentation, but it did not replace my own work.
